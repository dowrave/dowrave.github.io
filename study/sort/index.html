1. 삽입 정렬(Insertion Sort) $O(n^2)$

  • 맨 앞부터 현재까지의 부분 배열을 정렬함
  • 이미 정렬되어 있다면 $O(n)$
# 삽입 정렬
for i in range(1, len(arr)):
    for j in range(i, 0, -1):
        if arr[j] < arr[j-1]:
            arr[j], arr[j-1] = arr[j-1], arr[j]
  • range를 역순으로 구현하더라도 가장 마지막 인덱스(0)는 포함되지 않는 것만 주의하자.

2. 선택 정렬(Selection Sort) $O(n^2)$

  • 맨 앞부터 시작해서 현재 자리에 들어갈 최솟값을 찾는 정렬.
# 선택 정렬
for i in range(len(arr)):
	min_idx = i 
	for j in range(i+1, len(arr)):
		if arr[min_idx] > arr[j]:
			min_idx = j

	arr[i], arr[min_idx] = arr[min_idx], arr[i]

3. 버블 정렬(Bubble Sort) $O(n^2)$

  • 가장 큰 값을 맨 뒤로 보내는 정렬.
  • 효율이 매우 떨어짐 : Swap이 잦기 때문!
# 버블 정렬
for i in range(len(arr)):
    for j in range(1, len(arr) - i):
        if arr[j] < arr[j-1]:
            arr[j], arr[j-1] = arr[j-1], arr[j]
  • j=1부터 시작하는 것만 주의해주자. 0부터 시작하면 j-1은 배열의 가장 맨 뒤를 가리키게 된다.

4. 퀵 정렬(Quick Sort)

  • 분할 정복(Divide and Conquer)을 이용함
    • 주어진 문제를 작은 문제들로 분할해서 해결한 뒤, 그 결과들을 모아 큰 문제를 해결하는 방식.
  • 직접 구현하는 문제가 아니라면, 파이썬에서는 아래와 같이 이용할 수 있다.
    • arr.sort() : 배열 자체를 정렬함
    • sorted(arr) : 새로운 정렬된 배열을 반환함
# 퀵 정렬
def quick_sort(arr):
  """
  리스트를 새로 만들지 않게 하기 위해
  이런 식으로 구현함
  """
	def sort(low, high):
		if high <= low:
			return

		mid = partition(low, high)
		sort(low, mid - 1)
		sort(mid, high)

	def partition(low, high):
		pivot = (low + high) // 2 # 중간 "값"을 Pivot으로 선택
		while low <= high:
			while arr[low] < arr[pivot]:
				low += 1
			while arr[high] > arr[pivot]:
				high -= 1
			if low <= high:
				arr[low], arr[high] = arr[high], arr[low]
				low, high = low + 1, high - 1
		return low

	return sort(0, len(arr) - 1) # low, high는 여기서 인풋으로 정의됨

시간복잡도 : $nlogn$, $O(n^2)$

  • 기준 : 비교 횟수와 나누는 횟수
    • 비교 횟수 : $n$
    • 나누는 횟수
      • Best : $2^n$이 절반씩 쪼개지는 경우 : $logn$
      • Worst : 이미 정렬되어 있다면 나눠지는 기준은 항상 맨 앞이나 맨 뒤로 옴 : 데이터의 갯수인 $n$
  • 따라서
    • Best : $nlogn$
    • Worst : $O(n^2)$

5. 병합 정렬(Merge Sort)

  • 주어진 배열을 원소가 1이 될 때까지 계속 둘로 나눈 뒤, 나눠진 배열의 0번 인덱스부터 값을 비교해서 작은 값부터 새로운 배열에 넣어나가는 방식
  1. 이해하기 쉬운 버전
def merge_sort(arr):
    if len(arr) < 2:
        return arr
    
    mid = len(arr) // 2
    low_arr = merge_sort(arr[:mid])
    high_arr = merge_sort(arr[mid:])
    merged_arr = []
    
    low = high = 0
    

    while low < len(low_arr) and high < len(high_arr):
        if low_arr[low] < high_arr[high]:
            merged_arr.append(low_arr[low])
            low += 1
        else: # 같은 경우는 크게 상관 없을 듯
            merged_arr.append(high_arr[high])
            high += 1
    
    merged_arr += low_arr[low: ]
    merged_arr += high_arr[high: ]
    return merged_arr
  1. 메모리 효율 올린 버전(In-place sort)
def merge_sort(arr):
	def sort(low, high):
		if high - low < 2:
			return
		mid = (low + high) // 2
		sort(low, mid)
		sort(mid, high)
		merge(low, mid, high)

	def merge(low, mid, high):
		temp = []
		left, right = low, mid

		while left < mid and right < high:
			if arr[left] < arr[right]:
				temp.append(arr[left])
				left += 1
			else:
				temp.append(arr[right])
				right += 1

		while left < mid:
			temp.append(arr[left])
			left += 1
		while right < high:
			temp.append(arr[right])
			right += 1

		for i in range(low, high):
			arr[i] = temp[i - low]
	
	return sort(0, len(arr))

시간복잡도 : $O(nlogn)$

6. 카운팅 정렬

  • 양의 정수들에 대해, 수의 크기가 작고, 최대값과 최소값의 차이가 크지 않을수록 효과적이다.
  • 요점은 리스트의 인덱스를 해당 숫자처럼 활용하는 것이다 : 실제로 수들을 비교하는 게 아님!

ex) 5 3 3 1 2 4 3 3 1을 인풋inp으로 받았을 때, 이들을 오름차순으로 출력하고 싶다

  1. 최댓값까지의 인덱스를 갖는 리스트를 만든다. arr = [0] * max(inp)
  2. 들어오는 수 각각을 해당 인덱스에 저장한다.

1번째인 5를 넣은 결과는 [0, 0, 0, 0, 1]
3번째 원소까지 넣은 결과는 [0, 0, 2, 0, 1] …

  1. 0번째 인덱스부터, 값이 0이 아닌 경우 갖고 있는 값만큼 해당 인덱스를 출력한다.
