Hugo 깃허브 블로그 만들기

  • 이 블로그는 Hugo의 Mainroad 테마로 만들어졌고, 깃허브에 연결되어 있다.
  • 잘 설명된 글이 구글에 검색하면 바로 나오고, 따라가면 되는데 이상하게 깃허브랑 잇는 과정에서 자꾸 오류가 발생해서 정리해둔다.
  • 쉘 명령은 git bash를 이용했다. Hugo 자체에서 cmd나 Windows Powershell은 비추천하더라.

1. Git, Hugo 설치

2. 깃허브 레포지토리 생성

  • 하나는 myblog(이건 마음대로 지정)
  • 다른 하나는 <깃허브id>.github.io로 생성한다.(얘는 꼭 Public이어야 함)
  • 내 경우 레포지토리를 만들 때 둘 다add README.md를 켰다.

3. 프로젝트 만들기

hugo new site <프로젝트 이름>
  • 프로젝트 이름 폴더가 생성된다.

4. 프로젝트 폴더에서 깃 시작

cd <프로젝트 이름>
git init

5. 프로젝트를 myblog 레포지토리에 연결

git remote add origin <myblog 깃허브 레포지토리 URL>

6. 테마를 서브모듈로 다운받는다.

  • Hugo 테마 사이트
git submodule add origin <테마 깃허브 레포지토리 URL> themes/<테마 이름>
  • 테마 이름은 각 테마에 들어가면 잘 설명되어 있음.

7. config.toml 파일 수정

baseURL = 'http://<유저닉네임>.github.io`
...
theme = '<테마이름>'
  • baseURL하고 theme만 수정해주면 된다.

8. Public 폴더 삭제 및 명령 입력

  • 우선 Public 폴더를 삭제한다 : 안하면 아래 명령어가 안 먹음
git submodule add -b main <유저 닉네임.github.io 레포지토리 URL> public

9. 글 하나 생성

hugo new posts/my-first-post.md
  • content/posts/my-first-post.md 경로를 확인한다.

10. 서버 실행

hugo server -D
  • localhost:1313에 접속하거나 localhost:1313/posts/my-first-post에 접속해서 글이 있는지 확인한다.
  • 로컬호스트에서 실행하는 명령어이며, -D는 draft: true인 것들도 보여주는 명령어다.
  • 여기서 draft는 작업 중임을 의미한다.
    • 9.에서 만든 my-first-post.md의 윗부분(해당 영역을 프론트매터라고 한다.)을 보면 draft: true를 볼 수 있다.
    • 이게 있는 게시글은 아래에서 빌드할 때 올라가지 않는다. 작업이 끝났다면 제거해준다.

11. 종료 후 빌드

  • 종료는 hugo server를 실행한 쉘에서 ctrl + c를 눌러주자
hugo
# hugo -D 로 하면 draft: true도 올라가는 듯
  • 이 빌드 결과는 public에 저장되며, 위에서 서브모듈로 지정한 레포지토리로 push될 것이다.

12. 깃허브에 푸시

  • 레포지토리를 2개 관리하므로, 2개의 레포지토리 모두에 푸시한다.
# 1. 프로젝트 최상위 폴더에서 myblog에 푸시
git add .
git commit -m "<commit message>"
git push origin master

# 2. public 폴더 내용을 <닉네임>.github.io에 푸시
cd public
git add .
git commit -m "<commit message>"
git push origin main

  • 아래가 main인 이유는 8.에서 main으로 만들었기 때문이다.

  • 이후 <유저닉네임>.github.io에 들어가서 잘 올라갔는지 확인하면 끝!
