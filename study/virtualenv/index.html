[Python]Virtualenv를 통해 가상환경 설정하기

2023-05-22
가상환경

1. 과정

  • 모든 과정은 Windows Powershell로 진행했다.

1. Virtualenv 설치

pip install virtualenv

2. 가상환경 폴더 생성

virtualenv <프로젝트 폴더 이름>
  • virtaulenv은 venv와 달리 가상환경의 파이썬 버전도 지정할 수 있다.
virtualenv <프로젝트 폴더 이름> --python==3.9.13

3. 가상환경 활성화

cd <프로젝트 폴더 이름> # 프로젝트 폴더에 들어가기

/Scripts/activate # Scripts 폴더의 activate를 실행
  • 왼쪽에 <프로젝트 폴더 이름>이 뜨면 성공이다.

2. 설치된 모듈 이름과 버전 관리하기 : requirements.txt

  • 가상환경을 쓰는 이유는 프로그램을 사용하는 데 요구되는 패키지들의 버전을 일정하게 관리해주기 위함이다.
  • 이를 관리하는 텍스트 파일을 가상환경 내에 만들고 공유할 수 있다.

1. 패키지 관리 파일 생성하기

pip freeze > requirements.txt
  • 현재 가상환경에서 사용중인 모듈을 저장한다.

2. 패키지 관리 파일을 이용해 모듈들 설치하기

pip install -r requirements.txt

3. venv vs virtaulenv

  • 이 글의 내용을 가져오자면,
  1. venv는 virtaulenv에 비해 느리다.
  2. venv는 virtaulenv에 비해 확장성이 떨어진다.
  3. venv는 파이썬 버전을 다르게 해서 가상환경을 만들 수 없다.
  4. venv는 pip로 업그레이드할 수 없다.
  • 원 글에서도 3번을 중요시했고, 나도 3번 때문에 virutalenv를 설치했다. 현재 파이썬 버전은 3.10인데 아나콘다에서 실행한 버전은 3.9였기 때문에 버전을 맞춰줄 필요가 있었기 때문이다.
  • 가상환경이 필요할 때마다 자꾸 까먹어서 블로그에 정리해둔다.
