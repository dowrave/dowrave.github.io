랜덤 텔레포트와 랜덤 워크

2023-04-25
이론

선요약

  1. Random Walk는 단순히 이웃한 노드들 중에서 무작위로 선택해 이동하는 것을 의미한다.
  2. Random Teleport는 웹 페이지의 중요도가 수렴하지 않는 문제를 해결하기 위해 고안되었으며, 상황에 따라 연결되지 않은 노드로 이동할 수 있게끔 구성되었다.
  • KMOOC 데이터 마이닝
  • 기말고사 문제를 풀다가 당연히 맞다고 생각한 게 사실은 아니었어서 이참에 정리해둔다.

1. Random Walk

어떤 그래프가 있고, 매 시간 t마다 그래프 위의 랜덤한 노드에서 출발해서 이웃한 다른 노드로 가는 어떤 사람이 있다고 가정하자. 이 때 $P(t)$를 정의할 수 있으며, i번째 원소는 시간 t에서 사람이 i에 있을 확률을 나타낸다. t를 무한히 반복하게 되면, $P(t+1) = M^{T} \cdot P(t)$이다.

  • 이 때 $P(t)$를 Stationary Distribution이라고 한다.
  • $M$은 그래프의 인접행렬(Stochastic Adjecency Matrix)이며, $M^T$는 이를 벡터로 나타내기 위해 적용되었다.

그렇다면 $P(t)$는 어떤 점에서 출발하든 항상 같은 분포로 수렴할까?

Markov Process Theory

  1. 그래프의 어떤 점에서 시작하든, 다른 모든 점에 도달할 수 있어야 함
  2. 이분 그래프(Bipartite Graph)가 아니어야 함
  • 이분 그래프
  • 위의 조건을 만족하면 t를 무한으로 했을 때의 확률 분포는 어떤 점에서 출발하든 같은 값으로 수렴함이 알려져 있다.

2. Random Teleport

유래

  1. 검색 엔진에서 페이지의 중요도를 나타낼 필요가 있었음 : 검색어에 맞는 좋은 결과를 위로 띄울 수 있으니까
  2. 내 페이지의 중요도는 내 페이지로 향하는 링크를 건 다른 페이지의 중요도들로 결정됨(Flow Model, PageRank)
  3. Markov Process Theory를 만족한다면, 모든 페이지의 중요도는 어떤 값으로 수렴하게 되어 있음.
  4. 수렴하지 않는 경우가 2가지가 있음
  1. Dead-End : 받는 링크는 있는데, 이 페이지에서 나가는 링크가 없는 경우
  2. Spider-Trap : 그래프의 부분집합에서 나가는 링크가 부분집합을 벗어나지 않고 안에서 도는 경우
  • 이렇게 수렴하지 않는 경우를 해결하기 위한 방법으로 Random Teleport가 제시되었다.
  1. Dead-end : 다른 페이지로 향하는 링크(Out-link)가 없다면, 모든 페이지에 동일한 확률로 가도록 보정한다.
  2. Spider-Trap : 텔레포트 확률 c를 도입한다. 일반적으로 값은 0.1 ~ 0.2 정도라는 듯.
    • 1-c의 확률로 페이지에 있는 Out-link를 탄다.
    • c의 확률로 모든 페이지에 동일한 확률로 텔레포트된다.
